<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Copied PMTiles path -> vector tile provider | A local file path is opened by package code and kept alive by a provider/archive. |
| Theme metadata -> renderer | Theme URLs and style metadata can accidentally introduce network fetches or wrong source keys. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-04 | Tampering | PMTiles provider path | mitigate | Only use the Phase 1 copied/validated filesystem path; do not accept remote URLs in app code. |
| T-2-05 | Information Disclosure | Map style metadata | mitigate | Use V3/custom no-sprite theme path; tests reject remote sprite URLs and MapLibre/Mapbox package names. |
| T-2-06 | Denial of Service | Provider lifecycle | mitigate | Open provider once, close archive in dispose, and avoid rebuilding theme/provider every frame. |
</threat_model>
