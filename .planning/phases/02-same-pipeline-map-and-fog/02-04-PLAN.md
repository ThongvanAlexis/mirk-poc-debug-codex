Purpose: This is the phase's core proof surface. It must answer the architectural question in code: can map tiles and the atmospheric fog shader live in one Flutter map stack without MapLibre-style overlay compensation?

Output: Map + fog + seeded/in-memory reveal + blue dot + recenter + map-only toggle, with tests defending layer order and critical invariants.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Location/fix stream -> reveal state | External GPS-like data mutates the in-memory disc list and map camera targets. |
| Map camera -> fog shader uniforms | Camera state influences viewport, clip path, SDF, and shader draw in the render path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-10 | Tampering | Location fixes | mitigate | Accept only finite lat/lon values, use a fixed 25 m radius, and keep data in memory for POC scope. |
| T-2-11 | Denial of Service | Fog repaint/SDF rebuild | mitigate | Use ticker-driven paint for animation, cache SDF builds, and rebuild SDF only when disc/viewport key changes. |
| T-2-12 | Repudiation | Renderer mode evidence | mitigate | Map-only toggle remains visible/tested so UAT can separate vector tile cost from fog cost. |
</threat_model>
