<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User permission choice -> runtime state | OS permission results route the app to map or recovery states. |
| Platform location plugin -> reveal state | GPS plugin data becomes `GeoFix`, blue dot state, and in-memory reveal discs. |
| App file path -> platform share sheet | Active log path is passed to another app through share UI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-07 | Spoofing | Location fixes | mitigate | Reject non-finite/out-of-range coordinates at the adapter/repository boundary and log rejection reason. |
| T-3-08 | Information Disclosure | Share log | mitigate | Share only the active log file by explicit user action; no analytics or automatic upload. |
| T-3-09 | Denial of Service | Location stream lifecycle | mitigate | Stop subscriptions on dispose/lifecycle and keep foreground-only scope. |
</threat_model>
