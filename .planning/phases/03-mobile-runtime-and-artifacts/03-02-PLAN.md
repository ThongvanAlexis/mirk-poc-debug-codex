    - `kMaxLogsDirBytes` equals `10 * 1024 * 1024`.
    - Bootstrap creates `<documents>/logs/` and opens a UTC basic filename matching `YYYYMMDDTHHMMSSZ_logs.txt`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Logger API -> filesystem | In-process records become durable files in app documents storage. |
| Render/runtime callbacks -> log volume | High-frequency frame/map events can flood disk or jank the POC. |
| Failure path -> logger error handling | Logger write failures can recurse if handled through the logger itself. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-04 | Tampering | JSONL logs | mitigate | Use synchronous single-record writes and JSON encoding per line. |
| T-3-05 | Denial of Service | Log directory size | mitigate | Prune old logs to 10 MB and throttle frame/map event summaries. |
| T-3-06 | Denial of Service | Logger failures | mitigate | Catch only filesystem failures, use `developer.log`, and disable the active RAF. |
</threat_model>
