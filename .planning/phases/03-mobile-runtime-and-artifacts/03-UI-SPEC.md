# Phase 3 - UI Design Contract

> Visual and interaction contract for Phase 3 mobile runtime, foreground permission, diagnostics, and artifact support. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Flutter Material 3, manual contract |
| Preset | not applicable |
| Component library | Flutter Material widgets only |
| Icon library | Flutter Material `Icons` |
| Font | Platform default through `ThemeData`; no bundled custom font |

Design priority: preserve the map/fog evidence surface. UI elements must stay compact, readable, and out of the way of gesture/fps testing.

Sources: `.planning/phases/03-mobile-runtime-and-artifacts/03-CONTEXT.md`, `.planning/REQUIREMENTS.md`, `.planning/ROADMAP.md`, `.planning/research/FEATURES.md`, current `lib/main.dart`, current `lib/presentation/screens/map_screen.dart`.

## Screen Inventory

| Screen / State | Purpose | Primary Action | Secondary Action |
|----------------|---------|----------------|------------------|
| Permission rationale | Explain why foreground location is required before the system prompt. | `Enable Location` | none |
| Permission denied | Recover from denied or permanently denied foreground location. | `Open Settings` | `Check Permission` |
| PMTiles bootstrap loading | Show that local map data is being copied/opened before map launch. | none | none |
| PMTiles bootstrap error | Explain map-data startup failure and point to logs. | `Share Log` | `Retry Map Load` |
| Map runtime | Full-screen map/fog testing surface with compact controls. | map gestures | `Share Log`, `Recenter Map`, mode toggle |
| No GPS fix yet | Map is available but no accepted live fix has arrived. | `Recenter Map` disabled | keep logging active |

Out of scope: background location UI, notification permission UI, debug menu, historical log browser, full settings screen, production onboarding, and Phase 4 UAT result entry.

## Visual Hierarchy

| Surface | Focal Point | Hierarchy Contract |
|---------|-------------|--------------------|
| Permission rationale | `Enable Location` button | Title first, one short body paragraph second, primary button third. No illustration or decorative panel. |
| Permission denied | `Open Settings` button | Error heading first, recovery copy second, settings button third, check-again button fourth. |
| Map runtime | Full-bleed Melun map and fog | Map/fog content owns the screen. Controls are small SafeArea overlays and must never cover the center gesture area. |
| Map error/loading | State text | Short centered text, one recovery action if available. Avoid long diagnostic dumps in-app. |

Controls placement:

| Control | Placement | Notes |
|---------|-----------|-------|
| Map/Fog segmented mode toggle | top-right SafeArea, 12px inset | Keep existing `Map` and `Fog` labels; add semantic labels `Show map only` and `Show map with fog` if implementation changes touch semantics. |
| Share log action | bottom-left SafeArea, 12px inset | Icon button or small FAB with tooltip `Share active log`; do not use a full debug menu. |
| Recenter action | bottom-right SafeArea, 12px inset | Keep tooltip `Recenter to latest fix`; disabled until a valid latest fix exists. |

No visible UI text may explain internal implementation details such as SDF, shader uniforms, package names, keyboard shortcuts, or CI mechanics. Logs can contain those details.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dense inline separation |
| sm | 8px | Compact control padding, segmented control internal gap |
| md | 16px | Default form and state spacing |
| lg | 24px | Screen edge padding for permission/denied states |
| xl | 32px | Gap between state body copy and primary action |
| 2xl | 48px | Vertical spacing for centered state groups |
| 3xl | 64px | Reserved for rare full-screen state separation only |

Exceptions: icon-only touch targets must remain at least 44px square for mobile usability. This is a hit target rule, not a spacing token. Existing map overlay insets remain 12px because they are already established in `MapScreen` and align to the 4px grid.

---

## Typography

Use exactly these four sizes and two weights:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 | 1.3 |
| Body | 16px | 400 | 1.5 |
| Heading | 20px | 600 | 1.25 |
| Display | 24px | 600 | 1.2 |

Rules:

- Do not scale font size with viewport width.
- Letter spacing stays at the platform default; do not use negative letter spacing.
- Compact map controls use Label only.
- Permission and denied screen titles use Display only.

## Color

Use the MirkFall atmospheric palette plus the existing blue-dot accent:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1E2536` | Full-screen app background, dark map fallback, state-screen background |
| Secondary (30%) | `#3A4358` | Overlay surfaces, permission/denied state surface tint, disabled control surface |
| Accent (10%) | `#2B7CD6` | Primary CTA, active segmented mode, blue GPS dot, enabled recenter/share icon emphasis, focus indicators |
| Destructive | `#B3261E` | Destructive actions only; no destructive action is planned for Phase 3 |

Accent reserved for: `Enable Location`, `Open Settings`, active map/fog mode, latest-fix blue dot, enabled recenter control, enabled share-log control, and keyboard/accessibility focus indicators.

Supporting colors:

| Role | Value | Usage |
|------|-------|-------|
| Text primary | `#F4F7FB` | Text on dark dominant/secondary surfaces |
| Text muted | `#C7D0DF` | Secondary body copy and disabled explanatory text |
| Fog highlight | `#7C8AA3` | Non-interactive atmospheric accents only |

Do not use accent for every tappable element. Secondary actions use neutral surfaces with text/icon emphasis.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Enable Location` |
| Permission heading | `Enable foreground location` |
| Permission body | `The POC uses your current position to draw the blue dot, reveal 25 m fog discs, and write evidence logs for the renderer test.` |
| Denied heading | `Location is disabled` |
| Denied body | `Enable foreground location in system settings, then return to continue the Melun map test.` |
| Settings CTA | `Open Settings` |
| Permission recheck CTA | `Check Permission` |
| No-fix state heading | `Waiting for location` |
| No-fix state body | `Keep the app open with location enabled. The next accepted fix will reveal fog and enable recenter.` |
| Loading state | `Preparing Melun map` |
| Map opening state | `Opening Melun map` |
| Map-data error state | `Map data could not open. Restart the app or share the active log for diagnosis.` |
| Share-log CTA / tooltip | `Share active log` |
| Recenter tooltip | `Recenter to latest fix` |
| Destructive confirmation | none; Phase 3 has no destructive UI actions |

All button labels must be action-specific. Avoid generic confirmation, dismissal, and placeholder-empty-state copy.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Permission prompt trigger | Only `Enable Location` may trigger the system when-in-use permission request. Do not request permission automatically on app start before rationale copy is shown. |
| Permission granted | Proceed to PMTiles bootstrap and map launch; start foreground location stream through the runtime service seam. |
| Permission denied | Show denied state with `Open Settings` and `Check Permission`; keep share-log available if a log file exists. |
| App resume from settings | Recheck foreground permission and enter the map without requiring app restart when permission is now granted. |
| Share log | Invoke the platform share sheet for the active log file; log success, cancellation, and failure. |
| Recenter | Disabled until latest fix exists; when enabled, move to latest fix at zoom 15. |
| Map/Fog mode toggle | Preserve existing map-only versus map+fog measurement split. Toggle must not reset camera or latest fix state. |
| Loading/error recovery | Show short state text and a single recovery action when available. Long errors belong in logs. |

Accessibility:

- All icon-only controls require tooltips and semantic labels.
- Touch targets for icon-only controls must be at least 44px square.
- Text must remain legible on dark atmospheric surfaces; primary text contrast target is WCAG AA.
- Buttons must not resize the map overlay layout when enabled/disabled.

## Implementation Notes For Planner

- Prefer a small runtime shell around existing `PmtilesBootstrapScreen` and `MapScreen` rather than rewriting map/fog presentation.
- Keep plugin coupling outside `MapScreen`; adapt permission/location streams into `MapScreenServices.latestFixStream`.
- The share-log action belongs in the runtime/map UI as a compact control, not a new settings/debug menu.
- If the current `ThemeData(colorSchemeSeed: Colors.indigo)` is touched, replace it with an explicit `ColorScheme` derived from the color contract above.
- Loading/error copy in `main.dart` should be updated from `Preparing Melun PMTiles` and `PMTiles copy failed` to the copywriting contract above.
